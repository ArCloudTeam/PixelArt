# Pixel Frame Studio

A browser-based **pixel art and frame-by-frame animation editor**. The entire application is contained in a single `index.html` file built with plain HTML, CSS, and JavaScript—no frameworks, dependencies, or server required.

## Features

- Canvas sizes from `8 × 8` up to `64 × 64` pixels.
- Pencil, eraser, fill, and eyedropper tools.
- Color picker and built-in color palette.
- Toggleable pixel grid.
- Zoom controls without changing the artwork resolution.
- Undo and redo history with up to 60 states.
- Timeline for working with multiple animation frames.
- Create, duplicate, and delete frames.
- Animation playback from `1` to `24` FPS.
- Onion skin preview of the previous frame.
- Export the current frame as PNG.
- Export all frames as a horizontal PNG spritesheet.
- Save and load projects in JSON format.
- Mouse, touchscreen, stylus, and Pointer Events support.

## Quick Start

No installation is required.

1. Download or clone the project.
2. Open `index.html` in a modern web browser.
3. Choose a canvas size.
4. Draw your sprites and add frames from the timeline.
5. Export your work as a PNG or spritesheet.

You can also run the project using an optional local server:

```bash
python3 -m http.server 8080
```

Then open:

```text
http://localhost:8080
```

## Tools

| Tool | Description | Shortcut |
|---|---|---|
| Pencil | Paints a pixel using the selected color | `P` |
| Eraser | Clears a pixel | `E` |
| Fill | Fills a connected region of the same color | `F` |
| Eyedropper | Selects a color from the canvas | `I` |

### History

| Action | Shortcut |
|---|---|
| Undo | `Ctrl + Z` or `Cmd + Z` |
| Redo | `Ctrl + Shift + Z` or `Cmd + Shift + Z` |

## Frames and Animation

The timeline at the bottom of the editor allows you to:

- Add an empty frame after the current frame.
- Duplicate the selected frame.
- Delete the selected frame.
- Switch between frames using their thumbnails.
- Play or pause the animation.
- Set the playback speed from `1` to `24` frames per second.

The **Onion Skin** option displays the previous frame with reduced opacity, making it easier to create smooth frame-by-frame motion.

## Saving Projects

The **Save Project** option downloads a file named:

```text
pixel-frame-project.json
```

The file uses the following structure:

```json
{
  "version": 1,
  "gridSize": 16,
  "frames": [],
  "fps": 6
}
```

To continue working on a saved project, use **Load Project** and select a JSON file previously generated by the application.

> Changing the canvas size creates a new project and removes the current frames unless they were saved first.

## Exporting

### Current Frame PNG

Exports only the selected frame at its original resolution. For example, a `16 × 16` project generates a `16 × 16` image.

### PNG Spritesheet

Exports all frames in a single horizontal row.

For a `16 × 16` project with 4 frames, the spritesheet dimensions are:

```text
Width: 64 px
Height: 16 px
```

The horizontal position of each frame can be calculated with:

```text
x = frameIndex × canvasSize
```

## Current Limitations

- Spritesheets are exported horizontally only.
- Layers are not supported.
- Frames cannot be reordered with drag and drop.
- All frames use the same playback duration.
- Animated GIF and APNG export are not included.
- The history system stores complete project states, so large projects may use more memory.
